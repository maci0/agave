Fix Nemotron Nano MoE router overflow by implementing per-block scaling during router GEMV.

**Root cause:** Router weights are NVFP4 quantized. Current gemvNvfp4St dequantizes entire row to f32 before dot product, accumulating catastrophic quantization error (scores → -2.3e26).

**Solution:** Accumulate `(d_block * q_dot_block)` per quantization block, then sum scaled blocks.

**Changes to src/ops/quant.zig:**

Modify `gemvNvfp4St()` to accept a flag for per-block accumulation:

```zig
pub fn gemvNvfp4StPerBlock(
    x: [*]const f32,
    w_data: [*]const u8,   // NVFP4 nibbles (U32 tensor)
    scales: [*]const u8,   // FP8 E4M3 scales (U8 tensor)
    y: [*]f32,
    n: usize,
    k: usize,
) void {
    // NVFP4: 16 values per block, FP8 E4M3 scale per block
    const values_per_block = 16;
    const num_blocks = (k + values_per_block - 1) / values_per_block;

    for (0..n) |i| {
        var sum: f32 = 0.0;

        // Accumulate per-block: sum_blocks(scale_block * dot(q_block, x_block))
        for (0..num_blocks) |b| {
            const block_start = b * values_per_block;
            const block_size = @min(values_per_block, k - block_start);

            // Extract scale for this block (FP8 E4M3 → f32)
            const scale_idx = i * num_blocks + b;
            const scale = fp8e4m3ToF32(scales[scale_idx]);

            // Dequantize and dot product for this block only
            var q_dot: f32 = 0.0;
            for (0..block_size) |j| {
                const gi = block_start + j;
                const nibble = getNibble(w_data, i * k + gi);
                const q_val = nvfp4Lookup(nibble);  // 4-bit → f32 via LUT
                q_dot += q_val * x[gi];
            }

            // Accumulate scaled block result
            sum += scale * q_dot;
        }

        y[i] = sum;
    }
}
```

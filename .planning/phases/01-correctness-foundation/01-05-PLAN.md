# Plan 01-05: ROCm GPU Kernel Parity

## Objective

Implement ROCm GPU kernels for Q4_K, Q5_K, Q6_K, FP8 (E4M3/E5M2) GEMV and eliminate CPU fallbacks for sigmoidMul, fused SiLU, deinterleave, and DeltaNet. ROCm has the same quantization gaps as CUDA plus additional CPU fallbacks for elementwise ops. Port from CUDA PTX kernels (same algorithm, different ISA — AMDGCN uses `ds_swizzle` and `v_mov_dpp` for warp reductions instead of `shfl.sync.down`).

## Context

- ROCm test machine: `maci@192.168.0.205` (24GB VRAM)
- ROCm kernels live in `src/backend/kernels/rocm/` as Zig files compiled to AMDGCN via `zig build amdgcn`
- Existing ROCm kernels (f32, bf16, f16, q8_0, q4_0, mlx_q4, silu, gelu, add, mul, rmsNorm, softmax, l2Norm, rope, sdpa) provide reference patterns
- `src/backend/kernels/rocm/common.zig` has warp reduction helpers using `ds_swizzle_b32` and HIP intrinsics
- CUDA Q4_K/Q5_K/Q6_K/FP8 kernels (Plan 01-02) can be ported — same dequant logic, different reduction primitives

<action>
Implement 5 new ROCm GEMV kernels following the same pattern as existing `gemv_q4_0.zig` and `gemv_q8_0.zig`:

1. **gemv_q4_k.zig**: Port Q4_K dequant from CUDA kernel. Q4_K uses 256 values per super-block with nested sub-blocks (each 32 values with 6-bit scale + 4-bit min). Unpack 4-bit nibbles, apply per-sub-block scale and min, accumulate with warp reduction via `ds_swizzle_b32`.

2. **gemv_q5_k.zig**: Same structure as Q4_K but 5-bit values. Unpack using shift+mask (high bit from separate byte array `qh`, low 4 bits from `qs`). Apply per-sub-block scale and min.

3. **gemv_q6_k.zig**: 6-bit values with simpler structure (no nested sub-blocks). Unpack using `ql` (low 4 bits) + `qh` (high 2 bits). Apply per-block scale.

4. **gemv_fp8_e4m3.zig**: Use 256-entry comptime LUT (same as `fp8e4m3_lut` in `quant.zig`). Load u8, index into LUT, accumulate f32.

5. **gemv_fp8_e5m2.zig**: Same pattern but E5M2 format. Generate comptime LUT for E5M2 (5-bit exponent, 2-bit mantissa, bias=15).

All kernels must:
- Use warp reductions from `common.zig` (NOT block-level shared memory reductions)
- Dequantize in-kernel (NO pre-dequant to f32)
- Follow multi-row pattern if existing ROCm kernels use it
- Be registered in `all.zig` with the appropriate function names

Update `rocm.zig` gemv dispatch to route Q4_K, Q5_K, Q6_K, FP8_E4M3, FP8_E5M2 to GPU kernels instead of CPU fallback.

Update `build.zig` to include new kernel files in the AMDGCN build target.
</action>

<action>
Implement 3 ROCm GPU kernels to eliminate remaining CPU fallbacks:

1. **sigmoid_mul.zig**: In-place `data[i] *= sigmoid(gate[i])`. Simple element-wise kernel: `data[i] = data[i] * (1.0 / (1.0 + @exp(-gate[i])))`. Also add fused SiLU+multiply: `data[i] = data[i] * gate[i] * sigmoid(gate[i])`. Both follow same pattern as existing `silu.zig`.

2. **deinterleave.zig**: Separate interleaved pairs `[A0,B0,A1,B1...]` into `[A0,A1,...] [B0,B1,...]`. Simple index remapping kernel: `out_a[i] = in[2*i]`, `out_b[i] = in[2*i+1]`.

3. **deltanet.zig**: Port DeltaNet SSM recurrence from CPU. This is the most complex — has 4 sub-kernels matching Metal's approach: (1) causal conv1d with SiLU, (2) L2 norm, (3) decay computation, (4) state update + output. Use the GQA tiling pattern `h % n_kv` (NOT interleaved grouping).

Update `rocm.zig` to dispatch to GPU kernels instead of CPU fallback for `sigmoidMul`, `siluMul`, `deinterleave`, and `deltaNet`.

Register all new kernels in `all.zig`.
</action>

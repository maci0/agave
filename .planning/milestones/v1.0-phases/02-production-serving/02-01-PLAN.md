**TokenBucket struct** (one bucket for requests/min, one for tokens/min):

- `retryAfter(token_count: u32) u32`: Return `@max(request_bucket.retryAfterSeconds(1.0), token_bucket.retryAfterSeconds(@floatFromInt(token_count)))`.

Use `std.time.milliTimestamp()` for monotonic clock, convert to seconds via `@as(f64, @floatFromInt(ms)) / 1000.0`.

Create `src/scheduler.zig` implementing vLLM-style iteration-level continuous batching per RESEARCH.md Pattern 1.

**Request struct** (per client request):
- Fields: `id: u64`, `tokens: std.ArrayList(u32)`, `last_token_id: u32`, `is_finished: bool`, `is_cancelled: std.atomic.Value(bool)`, `enqueued_at: i64` (timestamp ms), `prompt_tokens: u32`
- `appendToken(token: u32, eog_ids: []const u32) void`: Append to tokens list, set last_token_id. Set is_finished if token matches any eog_id.
- `elapsedSeconds(now: i64) u32`: Return `@intFromFloat((@as(f64, @floatFromInt(now - enqueued_at)) / 1000.0))`.

**RequestManager struct** (request queue + active requests):
- Fields: `waiting: std.ArrayList(*Request)`, `running: std.ArrayList(*Request)`, `max_batch_size: usize`, `timeout_sec: u32`, `allocator: Allocator`, `mutex: std.Thread.Mutex`, `next_id: std.atomic.Value(u64)`
- `init(allocator: Allocator, max_batch_size: usize, timeout_sec: u32) !RequestManager`: Allocate waiting + running lists.
- `enqueue(prompt_tokens: u32) !*Request`: Lock mutex, allocate Request with unique ID, append to waiting, return pointer.
- `step(model: *Model, eog_ids: []const u32) !void`: Lock mutex during queue manipulation. Remove finished/cancelled from running. Check timeout on each request (`elapsedSeconds() > timeout_sec` → set is_cancelled). Fill batch from waiting (FIFO, up to max_batch_size - running.len). Unlock before model.forward(). For each running request: call `model.forward(req.last_token_id)`, append token via `req.appendToken()`.

**SchedulerStats struct** (for metrics):
- Fields: `waiting_count: u32`, `running_count: u32`, `completed_total: u32`, `cancelled_total: u32`

**Background scheduler loop helper** (not auto-started — server.zig controls lifecycle):
- Function: `runSchedulerLoop(manager: *RequestManager, model: *Model, eog_ids: []const u32, shutdown: *std.atomic.Value(bool)) void`: Loop `while (!shutdown.load(.acquire))`, call `manager.step(model, eog_ids)`, sleep 1ms between iterations.

Use `std.time.milliTimestamp()` for timestamps, `std.Thread.Mutex` for queue protection.

2. step() with max_batch_size=1 → 1 running, 1 waiting
3. step() after timeout_sec → request marked is_cancelled
4. step() after is_finished → request removed from running

fn validateAuth(server: *const Server, headers: []const u8) bool {
    if (server.api_key == null) return true; // No auth configured
    const needle = "authorization: bearer ";
    const idx = std.ascii.indexOfIgnoreCase(headers, needle) orelse return false;
    const token_start = idx + needle.len;
    const token_end = std.mem.indexOfScalarPos(u8, headers, token_start, '\r') orelse headers.len;
    const token = headers[token_start..token_end];
    return std.mem.eql(u8, token, server.api_key.?);
}
```

**Rate limit helper function** (add after validateAuth):

fn checkRateLimit(server: *Server, prompt_tokens: u32) ?u32 {
    if (server.rate_limiter == null) return null;
    if (server.rate_limiter.?.tryConsumeRequest(prompt_tokens)) return null;
    return server.rate_limiter.?.retryAfter(prompt_tokens);
}
```

    const body = "{\"error\":{\"message\":\"Invalid API key\",\"type\":\"invalid_request_error\",\"code\":\"invalid_api_key\"}}";

fn send429(stream: net.Stream, retry_after: u32) void {
    var buf: [256]u8 = undefined;
    const body = std.fmt.bufPrint(&buf, "{{\"error\":{{\"message\":\"Rate limit exceeded. Retry after {d} seconds.\",\"type\":\"rate_limit_exceeded\",\"code\":null}}}}", .{retry_after}) catch return;

    stream.writeAll(hdr) catch return;
    stream.writeAll(body) catch return;
}

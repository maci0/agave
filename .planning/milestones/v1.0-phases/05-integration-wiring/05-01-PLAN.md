<objective>
Activate the continuous batching scheduler in the HTTP server and wire RadixTree block reuse with actual block IDs from completed requests.

Purpose: Close two critical integration gaps (SERV-01, SERV-03) identified in the v1.0 milestone audit. Server currently bypasses the RequestManager scheduler (calling model.forward() directly under mutex) and RadixTree.insert() receives empty block lists. After this plan, HTTP requests flow through the scheduler and completed requests populate the RadixTree with real block IDs for prefix reuse.

Output: Modified server.zig (scheduler-routed), scheduler.zig (block extraction), model.zig (getBlockTable vtable method), all 6 model files (getBlockTable implementation)
</objective>

    2. Modify `generateStream()` (line ~1466) to route through scheduler instead of direct model.forward():
       - Remove the `g_server.mutex.lock(); defer g_server.mutex.unlock();` at the top
       - Instead of calling model.forward() directly for prefill and generate:
         a. Encode the prompt to token_ids (same as current)
         b. Call `g_server.request_manager.?.enqueue(token_ids)` to create a Request
         c. Poll `req.is_finished` or `req.is_cancelled` in a loop with 10ms sleep
         d. On each poll, check `req.tokens.items.len` for new tokens since last check, stream each new token via SSE
         e. When finished, send final chunk with finish_reason and `[DONE]`
         f. Call `req.deinit()` and `g_server.allocator.destroy(req)` in defer

       The key pattern for streaming from scheduler (replaces direct forward loop):
       ```zig
       const req = g_server.request_manager.?.enqueue(token_ids) catch {
           _ = sseWriteData(stream, "[DONE]");
           return;
       };
       defer {
           req.deinit();
           g_server.allocator.destroy(req);
       }

       var streamed_count: usize = 0;
       while (!req.is_finished and !req.is_cancelled.load(.monotonic)) {
           // Stream any new tokens
           const current_len = req.tokens.items.len;
           while (streamed_count < current_len) {
               const token_id = req.tokens.items[streamed_count];
               // Stream this token as SSE (same SSE formatting as current code)
               // ... (decode token, JSON escape, format chunk, sseWriteData)
               streamed_count += 1;
           }
           std.time.sleep(10_000_000); // 10ms poll
       }
       // Stream any remaining tokens after is_finished
       // Send final chunk with finish_reason="stop"
       // Send [DONE]
       ```
